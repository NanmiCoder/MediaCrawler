## 一、安全漏洞防护模式

### 1.1 路径遍历防护

**问题本质**：用户输入（URL 参数、请求体字段）直接拼入文件路径，`../` 等序列可跳出预期目录。

**修复模式**：

```python
from pathlib import Path

def validate_path_in_workspace(path: Path, workspace: Path) -> Path:
    """确保路径在 workspace 内，防止路径遍历"""
    resolved = path.resolve()
    workspace_resolved = workspace.resolve()
    if not resolved.is_relative_to(workspace_resolved):
        raise ValueError(f"路径越界: {resolved} 不在 {workspace_resolved} 内")
    return resolved
```

**关键要点**：
- 必须先 `resolve()` 再比较 — 符号链接、`..`、`.` 都会被规范化
- `is_relative_to()` 是 Python 3.9+ 的正确姿势，比手动字符串前缀检查安全
- 适用场景：API 接口中接受文件路径参数时、下载文件保存时、读取结果文件时

**本次修复**：#1（core.py:925 输出路径）、#3（routes.py:324 符号链接）、#5（core.py:560 下载路径）

### 1.2 SSRF 防护

**问题本质**：服务端根据用户输入发起 HTTP 请求，可访问内网服务。

**修复模式**：

```python
import ipaddress
import socket
from urllib.parse import urlparse

ALLOWED_SCHEMES = {"https", "http"}

def is_blocked_address(address: ipaddress.IPv4Address | ipaddress.IPv6Address) -> bool:
    return any((
        address.is_private,
        address.is_loopback,
        address.is_link_local,
        address.is_multicast,
        address.is_reserved,
        address.is_unspecified,
    ))

def validate_url(url: str) -> str:
    """校验 URL 防止 SSRF"""
    parsed = urlparse(url)
    if parsed.scheme not in ALLOWED_SCHEMES:
        raise ValueError(f"不允许的协议: {parsed.scheme}")
    hostname = parsed.hostname
    if not hostname:
        raise ValueError("URL 缺少主机名")

    # IP 字面量直接检查；域名必须解析并检查全部地址。
    try:
        addresses = {ipaddress.ip_address(hostname)}
    except ValueError:
        addresses = {
            ipaddress.ip_address(info[4][0])
            for info in socket.getaddrinfo(hostname, None)
        }

    if any(is_blocked_address(address) for address in addresses):
        raise ValueError(f"不允许访问非公网地址: {hostname}")
    return url
```

**关键要点**：
- 白名单优于黑名单 — 只允许 `https`/`http`
- 必须同时阻止私网、回环、链路本地、保留、组播和未指定地址
- 域名不能因为“不是 IP 字面量”而直接放行，必须检查全部解析结果
- 发起连接时还应绑定已验证地址或重新校验，降低 DNS rebinding 风险

**本次修复**：#2（core.py:951 下载 URL）

### 1.4 竞态条件写入

**问题本质**：先检查文件是否存在再创建，两步之间存在时间窗口，可被利用。

**修复模式**：

```python
import os

def safe_create_file(filepath: Path, content: bytes) -> bool:
    """原子创建文件，已存在则跳过"""
    try:
        fd = os.open(str(filepath), os.O_WRONLY | os.O_CREAT | os.O_EXCL, 0o644)
        try:
            os.write(fd, content)
        finally:
            os.close(fd)
        return True
    except FileExistsError:
        return False  # 已存在，安全跳过
```

**关键要点**：
- `O_CREAT | O_EXCL` 是操作系统级别的原子操作，比 `if not exists() then create()` 安全
- 适用于：锁文件创建、PID 文件、去重标记文件
- 不适用于需要覆盖写入的场景（用原子替换模式代替）

**本次修复**：#6（utils.py:112 竞态写入）

## 二、健壮性模式

### 2.1 原子文件写入

**问题本质**：写入中途崩溃导致文件截断、数据丢失。

**修复模式（通用）**：

```python
import tempfile
import os

def atomic_write(filepath: Path, write_func, *args, **kwargs):
    """
    原子写入文件。
    write_func: 接受文件对象，负责写入内容。
    """
    fd, tmp_path = tempfile.mkstemp(
        dir=str(filepath.parent),
        prefix=filepath.stem + ".tmp",
        suffix=filepath.suffix,
    )
    try:
        with os.fdopen(fd, "w", encoding="utf-8") as f:
            write_func(f, *args, **kwargs)
        os.replace(tmp_path, str(filepath))  # 原子替换
    except Exception:
        if os.path.exists(tmp_path):
            os.unlink(tmp_path)
        raise
```

**JSON 专用变体**：

```python
def atomic_write_json(filepath: Path, data: dict):
    fd, tmp_path = tempfile.mkstemp(
        dir=str(filepath.parent), suffix=".json"
    )
    try:
        with os.fdopen(fd, "w", encoding="utf-8") as f:
            json.dump(data, f, ensure_ascii=False, indent=2)
        os.replace(tmp_path, str(filepath))
    except Exception:
        if os.path.exists(tmp_path):
            os.unlink(tmp_path)
        raise
```

**CSV 专用变体**：

```python
def atomic_write_csv(filepath: Path, rows: list, fieldnames: list):
    fd, tmp_path = tempfile.mkstemp(
        dir=str(filepath.parent),
        prefix=filepath.stem + ".tmp",
        suffix=".csv",
    )
    try:
        with os.fdopen(fd, "w", encoding="utf-8-sig", newline="") as f:
            writer = csv.DictWriter(f, fieldnames=fieldnames, quoting=csv.QUOTE_ALL)
            writer.writeheader()
            writer.writerows(rows)
        os.replace(tmp_path, str(filepath))
    except Exception:
        if os.path.exists(tmp_path):
            os.unlink(tmp_path)
        raise
```

**关键要点**：
- 临时文件必须与目标文件在同一目录（`os.replace` 跨文件系统不可靠）
- 失败时清理临时文件，避免磁盘残留
- `os.replace()` 在 POSIX 和 Windows 上都是原子操作

**本次修复**：#20（CSV 原子写入）

### 2.2 可中断等待

**问题本质**：`time.sleep()` 阻塞线程，无法被外部信号中断（如取消请求）。

**修复模式**：

```python
import threading

class LongRunningService:
    def __init__(self):
        self._cancel_event = threading.Event()

    def cancel(self) -> None:
        """外部调用：取消正在执行的操作"""
        self._cancel_event.set()

    @property
    def is_cancelled(self) -> bool:
        return self._cancel_event.is_set()

    def _interruptible_sleep(self, seconds: float) -> bool:
        """可中断的等待。返回 True 表示被取消。"""
        return self._cancel_event.wait(timeout=seconds)

    def _do_work(self):
        # 替换 time.sleep(60)
        if self._interruptible_sleep(60):
            raise NonRetryableError("操作已被取消")
        # 继续正常逻辑...
```

**关键要点**：
- `Event.wait(timeout)` 在超时返回 `False`，在 `set()` 后返回 `True`
- 每个耗时循环开头检查 `cancel_event.is_set()`
- cancel 后抛不可重试异常，避免静默跳过

**本次修复**：#23（HTTP 429 sleep 不可中断）

### 2.3 OSError 不应静默

**问题本质**：`except OSError: pass` 吞掉异常，磁盘满、权限不足等严重问题无感知。

**修复决策树**：

```
OSError 发生
├── 操作可降级（如：读缓存文件、检查磁盘空间）
│   → 日志 warning/error + 降级处理（返回空数据/False）
│   → 例：#8 check_disk_space → 返回 False
│   → 例：#10 去重索引加载 → 返回 {} 但记 error
│
├── 操作不可降级（如：写入关键数据、磁盘满时继续写）
│   → 日志 error + 抛异常（FatalError / ConfigError）
│   → 例：#9 check_disk_space_enforced → FatalError
│   → 例：#16 int() 转换失败 → ConfigError
│
└── 日志系统本身失败（如：JSONL 写入 OSError）
    → 降级到 Python logging（写 stderr）
    → 例：#25 write_log → logging.warning()
```

**关键要点**：
- 绝不 `pass` — 至少记录日志
- 降级处理要明确语义：`False` 表示"检查失败，假设不安全"而非"没问题"
- 日志系统本身的失败走 stderr，不写文件

**本次修复**：#8, #9, #10, #25

### 2.4 TOCTOU 竞态防护

**问题本质**：先检查条件再操作，两步之间状态可能改变。

**修复模式**：

```python
# ❌ 错误：exists() 和 stat() 之间文件可能被删除
if not (video_file.exists() and video_file.stat().st_size > 10000):
    download(video_file)

# ✅ 正确：直接操作，处理异常
try:
    video_size = video_file.stat().st_size
except FileNotFoundError:
    video_size = 0
if video_size <= 10000:
    download(video_file)
```

**关键要点**：
- 优先 EAFP（Easier to Ask Forgiveness than Permission）而非 LBYL
- 检查和操作应尽量合并为一步原子操作
- 无法合并时（如检查和操作跨系统调用），至少处理中间异常

**本次修复**：#11（core.py:606）、#6（utils.py:112）

### 2.5 输入校验（Pydantic）

**问题本质**：API 参数无校验，非法输入导致运行时错误或安全漏洞。

**修复模式**：

```python
from pydantic import BaseModel, Field, field_validator
from typing import List, Literal

class SearchRequest(BaseModel):
    # 1. 字段级约束
    keywords: List[str] = Field(..., description="关键词列表")
    max_count: int = Field(20, ge=1, le=200)

    # 2. 自定义校验器
    @field_validator("keywords")
    @classmethod
    def validate_keywords(cls, v: List[str]) -> List[str]:
        if not v:
            raise ValueError("keywords 不能为空")
        if len(v) > 50:
            raise ValueError("keywords 最多 50 个")
        for kw in v:
            if len(kw) > 200:
                raise ValueError(f"关键词过长: {kw[:50]}...")
        return v

class ScriptsRequest(BaseModel):
    # 3. 枚举约束（比字符串安全得多）
    model: Literal["tiny", "base", "small", "medium", "large"] = Field("small")
```

**校验层次**：

| 层 | 工具 | 示例 |
|----|------|------|
| 类型约束 | `Literal`, `int`, `List[str]` | model 枚举 |
| 范围约束 | `Field(ge=, le=)` | max_count 1-200 |
| 业务约束 | `@field_validator` | keywords 长度、空列表 |
| 跨字段约束 | `@model_validator` | 日期范围 |

## 三、运维健壮性模式

### 3.1 daemon 线程 + 优雅退出

**问题本质**：`daemon=True` 线程在进程退出时被强杀，任务状态永远卡在 "running"。

**修复模式**：

```python
import atexit

class TaskManager:
    def __init__(self):
        # ❌ daemon=True → 进程退出时线程被杀，任务卡 running
        # ✅ daemon=False + atexit 清理
        atexit.register(self._on_exit)

    def _on_exit(self):
        """进程退出时将所有 running 任务标记为 failed"""
        for task in self._tasks.values():
            if task.status == "running":
                task.status = "failed"
                task.error = "进程退出时任务仍在运行"
        self._save_registry()

    def cleanup_stale_tasks(self, max_age_hours: int = 1):
        """清理超过 N 小时仍在 running 的任务（上次异常退出遗留）"""
        now = time.time()
        for task in self._tasks.values():
            if task.status == "running":
                age = now - task.created_at
                if age > max_age_hours * 3600:
                    task.status = "failed"
                    task.error = f"任务运行超 {max_age_hours} 小时，标记为 failed"
        self._save_registry()
```

**关键要点**：
- `atexit` 确保正常退出时状态正确
- `cleanup_stale_tasks` 处理异常退出（kill -9、断电等 atexit 来不及跑的情况）
- 启动时自动调用 `cleanup_stale_tasks`

**本次修复**：#12

### 3.2 任务删除清理磁盘

**问题本质**：删除任务只删记录，workspace 目录残留，长期运行磁盘满。

**修复模式**：

```python
import shutil

def delete_task(self, task_id: str) -> bool:
    with self._lock:
        if task_id in self._tasks:
            task = self._tasks[task_id]
            # 先清理磁盘（删记录前）
            workspace_path = Path(task.workspace)
            if workspace_path.exists():
                try:
                    shutil.rmtree(str(workspace_path))
                except Exception as e:
                    logger.warning("清理 workspace 失败: %s: %s", workspace_path, e)
                    return False
            del self._tasks[task_id]
            self._save_registry()
            return True
```

**关键要点**：
- 先删文件再删记录 — 如果先删记录，删文件失败时无法重试
- 不要同时使用 `ignore_errors=True` 又期望捕获异常；它会吞掉清理失败
- 清理失败时保留任务记录并返回失败，便于定位和重试

**本次修复**：#22

### 3.3 进程管理：terminate → wait → kill

**问题本质**：`terminate()` 后不等进程退出，僵尸进程残留。

**修复模式**：

```python
def kill_process_on_port(port: int) -> bool:
    for proc in psutil.process_iter(["pid", "name"]):
        try:
            for conn in proc.connections():
                if conn.laddr.port == port:
                    proc.terminate()
                    try:
                        proc.wait(timeout=5)
                    except psutil.TimeoutExpired:
                        proc.kill()
                        try:
                            proc.wait(timeout=3)
                        except psutil.TimeoutExpired:
                            logger.warning("进程 PID=%d kill 后仍未退出", proc.pid)
                    return True
        except (psutil.NoSuchProcess, psutil.AccessDenied):
            continue
    return False
```

**关键要点**：
- `terminate()` → `wait(timeout=5)` → `kill()` → `wait(timeout=3)`
- 逐步升级，给进程优雅退出的机会
- 最终仍无法退出时记 warning（不要无限等待）

**本次修复**：#28

## 四、代码质量模式

### 4.1 异常信息不泄露内部路径

**问题本质**：`str(e)` 可能包含文件路径、堆栈、SQL 语句等内部信息。

**修复模式**：

```python
from douyin_scraper.exceptions import ScraperError

@app.exception_handler(Exception)
async def handle_exception(request, exc):
    if isinstance(exc, ScraperError):
        # 业务异常：返回详细信息（我们控制的消息）
        return JSONResponse(status_code=400, content={"error": str(exc)})
    else:
        # 未预期异常：返回通用消息，详情只写日志
        logger.error("未预期异常: %s", exc, exc_info=True)
        return JSONResponse(status_code=500, content={"error": "内部错误，请查看日志"})
```

**关键要点**：
- 区分"我们抛的异常"和"系统异常"
- 业务异常的消息是我们写的，安全可控
- 系统异常可能包含路径、SQL、密钥等，绝不直接暴露
- `exc_info=True` 确保完整堆栈写入日志

**本次修复**：#21

### 4.2 Path 对象的 truthy 陷阱

**问题本质**：`Path` 对象总是 truthy，`not Path("")` 返回 `False`。

```python
# ❌ 永远不会进入 if 分支
if not self.project_dir:
    errors.append("project_dir 未设置")

# ✅ 正确检查
if self.project_dir is None or not str(self.project_dir).strip():
    errors.append("project_dir 未设置")
```

**Python truthy 陷阱清单**：

| 表达式 | 结果 | 原因 |
|--------|------|------|
| `bool(Path(""))` | `True` | Path 对象总是 truthy |
| `bool(Path("."))` | `True` | 同上 |
| `not Path("")` | `False` | 同上 |
| `bool(datetime.time(0, 0))` | `True` | 现代 Python 中 `time` 对象始终为 truthy |
| `bool(numpy.float64(0.0))` | `False` | `bool(numpy.float64("nan"))` 为 `True`，不能用 truthy 判断有效数值 |

**本次修复**：#24

### 4.6 深度限制的文件搜索

**问题本质**：`rglob("*.csv")` 递归搜索所有子目录，大目录很慢。

```python
# ❌ 无限制递归
csv_files = list(workspace.rglob("*.csv"))

# ✅ 限制深度为 2 层
csv_files = [
    p for p in workspace.rglob("*.csv")
    if len(p.relative_to(workspace).parts) <= 2
]
```

## 五、并发模式

### 5.1 串行转并发的安全改造

**原则**：默认串行（`max_workers=1`），显式配置后才并发。

```python
from concurrent.futures import ThreadPoolExecutor, as_completed

max_workers = getattr(self._config, "max_workers", 1)
if not isinstance(max_workers, int) or max_workers < 1:
    max_workers = 1

if max_workers <= 1:
    # 串行模式（保持原有行为）
    for i, rec in enumerate(valid, 1):
        _process_one(i, rec)
else:
    # 并发模式
    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        futures = {}
        for i, rec in enumerate(valid, 1):
            if self._cancel_event.is_set():
                break
            future = executor.submit(_process_one, i, rec)
            futures[future] = i
        for future in as_completed(futures):
            try:
                result = future.result()
            except Exception as e:
                logger.error("并发处理异常: %s", e)
```

**线程安全检查清单**：
- [ ] 共享状态有锁保护（`_lock`、`threading.Lock`）
- [ ] 文件写入是原子的（`append_record` 内部有原子写入）
- [ ] 磁盘检查仅主线程执行（避免 N 个线程同时报磁盘满）
- [ ] 取消检查每条记录执行（`_cancel_event.is_set()`）
- [ ] 日志系统本身线程安全（`logging` 模块线程安全）
